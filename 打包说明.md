# APK 打包指南

本项目提供 **3 种打包方式**，按推荐度从高到低排列。

---

## ✅ 方式一：GitHub Actions 云端构建（推荐，零配置）

> 不需要安装任何软件，完全在云端自动构建，免费。

### 步骤

1. **将 TxtReader 项目推送到 GitHub**

   ```bash
   cd TxtReader
   git init
   git add .
   git commit -m "feat: TXT小说阅读器初始版本"
   # 在 github.com 创建一个新仓库，然后：
   git remote add origin https://github.com/你的用户名/TxtReader.git
   git push -u origin main
   ```

2. **触发构建**
   - 推送后，GitHub 自动运行 `.github/workflows/build-apk.yml`
   - 也可以在 GitHub → Actions 页面 → 手动点击 `Run workflow`

3. **下载 APK**
   - 等待约 **3~5 分钟**
   - 进入 GitHub → Actions → 点击最新的 workflow run
   - 在 **Artifacts** 区域下载 `TxtReader-debug-apk.zip`
   - 解压后得到 `app-debug.apk`，直接安装到手机即可

## ✅ 方式二：Android Studio 本地构建（最稳定）

### 1. 安装 Android Studio
- 下载地址：https://developer.android.com/studio
- 安装时勾选：Android SDK、Android Virtual Device
- 安装完成后，Android Studio 会自动下载 SDK 到 `C:\Users\你的用户名\AppData\Local\Android\Sdk`

### 2. 打开项目
- 启动 Android Studio
- 选择 **Open** → 选择 `TxtReader/` 文件夹（不是 `TxtReader/app`）
- 等待 Gradle 同步（首次约 5~10 分钟，需要下载依赖）

### 3. 构建 APK
- 菜单栏：**Build** → **Build Bundle(s) / APK(s)** → **Build APK(s)**
- 等待构建完成（右下角进度条）
- 完成后点击提示框中的 **locate** 链接
- APK 位置：`TxtReader\app\build\outputs\apk\debug\app-debug.apk`

## ✅ 方式三：命令行构建（需手动安装环境）

### 需要安装
1. **JDK 17**：https://adoptium.net/（选 Temurin 17 LTS）
2. **Android SDK Command-line Tools**：
   https://developer.android.com/studio#command-tools
3. 设置环境变量：
   ```powershell
   [System.Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Eclipse Adoptium\jdk-17.x.x-hotspot", "User")
   [System.Environment]::SetEnvironmentVariable("ANDROID_HOME", "$env:LOCALAPPDATA\Android\Sdk", "User")
   ```
4. 安装 SDK 组件：
   ```powershell
   sdkmanager "platform-tools" "platforms;android-34" "build-tools;34.0.0"
   ```

### 构建命令
```powershell
cd TxtReader
.\gradlew.bat assembleDebug
# APK 输出：app\build\outputs\apk\debug\app-debug.apk
```

---

## 📱 安装到手机

构建成功后，将 `app-debug.apk` 传输到手机安装：

1. 手机「设置」→「安全」→ 开启 **「允许未知来源」** 或 **「安装未知应用」**
2. 用文件管理器找到 APK 文件，点击安装
3. 或通过 ADB 安装：`adb install app-debug.apk`

## ⚠️ 常见问题

| 问题 | 解决方案 |
|------|----------|
| Gradle 同步失败 | 检查网络，或在 gradle.properties 中配置代理 |
| `SDK not found` | 在 Android Studio 的 SDK Manager 中下载 SDK 34 |
| 安装时提示"解析错误" | 确认手机 CPU 架构（ARM64），debug APK 默认包含全架构 |
| `minSdk` 不兼容 | 当前 minSdk=24，需要 Android 7.0+ 的手机 |
